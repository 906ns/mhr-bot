# モンハンサンブレイク 傀異化素材検索 Discord bot を作って

## 概要
Discord上で素材名を打つと、その傀異化素材を入手できるモンスター名とURLを返すbotを作りたい。素材に複数の系統がある場合は、ボタンで対話的に選ばせる。

## 動作仕様
- ユーザーがチャンネルに素材名（例:「爪」）を打つ
- 素材のレア度（爪/尖爪/剛爪 など）は入手元モンスターが同じなので区別しない。素材の「種類」だけで持つ
- ただし「通常系」と「凶系」は入手元モンスターが異なるので、系統として分ける
  - 例:「爪」(通常系=爪/尖爪/剛爪) と「凶系」(凶爪/凶尖爪/凶剛爪) は別モンスター
- 打たれた素材に系統が複数あれば、botがボタンを出して「どの系統か」を選ばせる（対話形式）
- 系統が1つしかない素材なら、ボタンを出さず即座に結果を返す
- ボタンが押されたら、その系統のモンスター名とURLを返信する
- 出力はモンスター名とURLのみ（弱点属性や難易度などは不要）
- 該当する素材がなければ「見つかりませんでした」と返す
- 素材名にヒットしない通常の発言には反応しない（黙る）

## 出力イメージ
```
ユーザー: 爪
bot: 「爪」のどの系統を調べますか？
     [通常系] [凶系]   ← ボタン
ユーザー: （通常系を押す）
bot: 【爪（通常系）】
     ゴシャハギ https://appmedia.jp/mhrise/6123547
     オロミドロ https://appmedia.jp/mhrise/6123541
```

## 技術要件
- 言語: Python
- ライブラリ: discord.py（ボタンは discord.ui.View / discord.ui.Button を使う）
- トークンは .env から読む（python-dotenv を使用）。コードに直書きしない
- .gitignore を作り、.env を必ず除外する
- message_content インテントを有効にする

## データ構成
データは data.py に Python の辞書として持つ。URLは正規化し、モンスター名→URLは1箇所だけで管理して重複させないこと。

```python
# モンスター名 → URL（1箇所で管理）
MONSTERS = {
    "ゴシャハギ": "https://appmedia.jp/mhrise/6123547",
    "オロミドロ": "https://appmedia.jp/mhrise/6123541",
    "ヤツカダキ": "https://appmedia.jp/mhrise/6123532",
    "イソネミクニ亜種": "https://appmedia.jp/mhrise/75697477",
    "ラージャン": "https://appmedia.jp/mhrise/6129847",
    "ヤツカダキ亜種": "https://appmedia.jp/mhrise/75777412",
}

# 素材の種類 → 系統 → そのモンスター名リスト
MATERIALS = {
    "爪": {
        "通常系": ["ゴシャハギ", "オロミドロ", "ヤツカダキ", "イソネミクニ亜種"],
        "凶系": ["ラージャン", "ヤツカダキ亜種"],
    },
}
```
※データは「爪」系だけサンプルで入れておけばOK。残りの素材（骨/皮/牙/鱗/甲殻/血/竜骨など）は私が後で同じ形式で追記する。

## 設計上の指示
- 「素材を探して系統を返す処理」と「返信メッセージを組み立てる処理」を別の関数に分ける（関心の分離）。後でWeb版やLINE版に流用したいため
- bot自身の発言には反応しないようにして無限ループを防ぐ
- 系統が1つだけの素材と複数ある素材の両方に対応する
