# 既存の傀異化素材検索 Discord bot を改修してほしい

すでに動いている Discord bot がある（bot.py / data.py）。これを以下の仕様に改修してほしい。
data.py は新しい構造のものに差し替え済みなので、その構造に合わせて bot.py を書き直すこと。

## 重要: data.py の新しい構造
data.py は以下の構造に変わっている（すでに差し替え済み・中身は編集しないこと）:

```python
MONSTERS = {
    "アオアシラ": "https://mhrise.com/data/4446.html",
    # ... モンスター名 → URL
}

MATERIALS = {
    "骨": {
        "yomi": ["ほね"],                                              # ひらがな読み(複数可)
        "tiers": ["傀異化した骨", "傀異化した堅骨", "傀異化した重骨"],  # レア度3段階の素材名
        "monsters": ["アオアシラ", "ラングロトラ", "ウルクスス"],        # 入手元モンスター
    },
    "凶骨": { ... },
    "竜骨": { ... },
    "龍骨": { ... },
    # ... 全素材カテゴリ
}
```

つまり各素材カテゴリは "yomi"(ひらがな読みのリスト) と "tiers"(素材名3段階のリスト) と "monsters"(モンスター名のリスト) を持つ。
さらに一部のカテゴリは "alias"(漢字の別名リスト) を持つ（例: "殻" は alias に "甲殻" を持つ）。
"alias" と "yomi" は存在しないカテゴリもあるので、コードでは .get("alias", []) のように
「無ければ空リスト」として安全に扱うこと。

## 動作仕様

### 1. 入力と部分一致での候補抽出（漢字・ひらがな両対応）
- ユーザーが素材名（例:「骨」または「ほね」）をチャンネルに打つ
- 入力文字を、各カテゴリの **キー(漢字) または yomi(ひらがな読み) に部分一致** で照合し、
  当てはまるカテゴリをすべて拾う
  - 漢字の例:「骨」と打つと "骨" を含む 骨 / 凶骨 / 竜骨 / 龍骨 の4カテゴリがヒット
  - ひらがなの例:「つめ」と打つと yomi に "つめ" を持つ 爪 / 凶爪 の2カテゴリがヒット
  - 「竜骨」「りゅうこつ」なら 竜骨 等、読みでも同様に動く
- 照合ロジック: あるカテゴリについて、照合対象を
  「カテゴリ名 + そのカテゴリの alias(あれば) + yomi(あれば)」の文字列リストにまとめ、
  そのいずれかに入力が部分一致(in)すればヒットとする
  - 例:「甲殻」と打つと、"殻" カテゴリの alias に "甲殻" があるのでヒットする
- ヒット0件なら反応しない（黙る）

### 2. 候補が複数なら選択ボタンを出す（対話形式）
- ヒットしたカテゴリが2件以上なら、discord.ui の Button でカテゴリを選ばせる
  - 例:「骨」→ [骨] [凶骨] [竜骨] [龍骨] の4ボタン
- ヒットが1件だけなら、ボタンを出さず即座に結果（後述）を出す

### 3. カテゴリ確定後の出力（チャンネル）
選ばれた（または1件確定した）カテゴリについて、以下のフォーマットで出力する:

```
傀異化した骨
傀異化した堅骨
傀異化した重骨
（ここで1行あける）
アオアシラ
ラングロトラ
ウルクスス
```

- 1〜3行目: そのカテゴリの "tiers"（素材名3段階）を3行で
- 1行あける
- 続けて "monsters"（入手元モンスター名）を改行区切りで列挙
- **この段階ではURLは出さない**（モンスター名だけ）

### 4. リアクションでURLをDM送信
- 上記の結果メッセージに、bot が 📩 のリアクションをあらかじめ1つ付けておく
- ユーザーがその 📩 を押したら、そのカテゴリの入手元モンスター全員ぶんのURLを、
  **押したユーザーの個人DM** にまとめて送る
- DMの中身は「モンスター名 + URL」を改行区切りで列挙する。例:
  ```
  アオアシラ https://mhrise.com/data/4446.html
  ラングロトラ https://mhrise.com/data/4490.html
  ウルクスス https://mhrise.com/data/4456.html
  ```
- 📩 以外のリアクションには反応しない
- bot 自身が付けた初期リアクションに反応してDMを送らないよう、リアクションしたuser が bot 自身なら無視する
- どのメッセージがどのカテゴリの結果なのか（=どのモンスター群のURLを送るか）を
  bot 側で覚えておく必要がある。メッセージID → モンスター名リスト の対応を保持する仕組みを作ること

## この仕様が全素材に適用されること
上記は「骨」を例にしているが、爪・牙・鱗・血・甲殻・皮・竜骨・角・翼膜・龍骨・龍血など
**すべての素材カテゴリで同じ動作**になること。素材ごとに個別のコードを書くのではなく、
MATERIALS のデータを読んで汎用的に動くロジックにすること。

## 技術要件
- 言語: Python / discord.py
- リアクションを扱うため、reactions インテントと message_content インテントを有効にする
- DM送信のため、メンバーへの送信を行う（送信に失敗した場合=DM拒否設定などは、
  チャンネルに「DMを送れませんでした」と短く通知する形でフォロー）
- トークンは .env から読む（既存のままでよい）

## 設計上の指示
- 「入力から候補カテゴリを探す処理」「結果メッセージを組み立てる処理」「DM用のURL一覧を組み立てる処理」を
  それぞれ別の関数に分ける（関心の分離）
- メッセージID→モンスターリストの対応は、まずはメモリ上の辞書で持てばよい
  （bot再起動で消えてよい。永続化は不要）
- bot 自身の発言・リアクションには反応しない（無限ループ防止）
